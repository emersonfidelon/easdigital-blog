Wildcards, subdomínios dinâmicos na VPS

Se você desenvolve sistemas para web e nunca teve que publicar algo em uma VPS, tenha certeza que um dia terá que fazer isso, assim espero.

Até o ponto de publicar o sistema em uma VPS tudo certo, já havia feito isso antes, não tenho dificuldade em trabalhar com o terminal do linux, até prefiro, então qual a motivação desse post?

A questão é que o projeto era em wordpress e usava a função multisite, se você não sabe o que é wordpress ou não conhece a função multisite é só clicar aqui.

O multisite consiste em poder criar vários sites dentro de uma única instalação do wordpress, podendo ser instalado em subdiretório:

Ou subdomínio:

Chegamos então no motivo deste post, todo projeto foi criado com base em subdomínio, como fazer para que o DNS respondesse a qualquer solicitação no domínio principal? Essa é a função do wildcard (coringa).

Exemplo:

Porém o DNS era resolvido pelo Registro.br que não aceita a entrada wildcard. A melhor alternativa que encontrei foi usar o PointDNS, que atende a solicitação de dominio coringa, e além de tudo não tem custo.

Domínio devidamente registrado, conta criada no PointDNS, hora das configurações. Crie uma entrada para o domnio que deseja mapear.

No PointDNS coloque todas as entradas do tipo A apontando para o IP da VPS, Assim como um ALIAS apontando para seu domínio, um CNAME com WWW e outro contendo seu wildcard *.exemplo.com.br (Lembre-se de mudar exemplo pelo seu domínio).

As entradas NS são criadas pelo próprio PointDNS, elas é que serão usadas no Registro.br, aqui nós já acabamos, vamos para lá.

Já no Registro.br depois de entrar com seu usuário e senha e clicar sobre o domínio que deseja editar, mesmo domínio que mapeou no PointDNS, basta clicar em ALTERAR SERVIDORES DNS e inserir as entradas NS criadas pelo PointDNS. Insira as entradas em ordem crescente.

Pronto, agora toda vez que seu dominio ou subdomínio for requisitado o Registro.br vai encaminha a requisição para o PointDNS que se encrregará de reconhecer a chamada e resolver para destino mais apropriado.

Sabe de uma maneira melhor para resolver essa questão? Resolveria assim porém com uma abordagem ainda mais simples? Conte-nos tudo aí nos comentários. Vamos aprender juntos.
